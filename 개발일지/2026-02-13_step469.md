# Step 469 개발일지

- 작성시각: 2026-02-13 19:10:23
- 해결하고자 한 문제:
  - 홈 인트로 배경 이미지(고용량 PNG)가 늦게 렌더되는 체감 성능 문제.
  - 결과/시뮬/최종 화면에서 가이드 프로필 이미지가 지연 표시되는 문제.

- 해결된 것:
  - `Home.tsx`
    - 첫 배경 이미지(`home1`) 선로딩 완료 전에는 인트로 진행을 시작하지 않도록 제어.
    - 홈 배경 + 카테고리 배경 이미지를 `Image()`로 병렬 프리로드.
    - 초기 로딩 중에는 어두운 그라디언트 fallback 화면을 표시해 깜빡임/빈 화면 체감 완화.
  - `main.tsx`
    - 가이드 이미지 4종(`agent_han/song/choi/you`)을 앱 시작 시점에 eager preload.
    - 동시에 `<link rel="preload" as="image">`를 head에 주입해 네트워크 우선순위 상향.
  - 검증
    - `pnpm eslint src/pages/Home.tsx src/main.tsx` 통과.

- 해결되지 않은 것:
  - 원본 이미지 파일 용량 자체가 커서(수 MB), 네트워크가 느린 환경에서는 한계가 있음.
  - 다음 단계에서 WebP/AVIF 변환 및 해상도별 분기(srcset) 적용 시 추가 개선 가능.
