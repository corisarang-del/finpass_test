# 개발일지 - 2026-02-11 (여백 문제 근본 해결)

작성시각: 2026-02-11 17:24

## 해결하고자 한 문제
1. 캐시를 삭제했는데도 파비콘이 나타남
2. Grid 레이아웃으로 변경했는데도 카테고리 이미지에 여백이 남음

## 해결된 것
### 1. 파비콘 관련
- public/vite.svg 이미 삭제 완료
- 파비콘은 브라우저 자체 캐시 또는 개발 서버 재시작 필요
- 개발 서버 완전 재시작 후 확인 필요

### 2. 여백 문제의 근본 원인 발견
**App.css의 #root 스타일이 문제였음!**

#### 변경 전 (App.css)
```css
#root {
  max-width: 1280px;  /* ← 최대 너비 제한 */
  margin: 0 auto;      /* ← 중앙 정렬로 양쪽 여백 생성 */
  padding: 2rem;       /* ← 상하좌우 패딩 */
  text-align: center;
}
```

#### 변경 후 (App.css)
```css
#root {
  width: 100%;
  height: 100vh;
  margin: 0;
  padding: 0;
}
```

### 3. body 스타일 추가 (index.css)
```css
body {
    margin: 0;
    padding: 0;
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
}
```

### 4. index.html 개선
- `lang="en"` → `lang="ko"` 변경

## 기술적 분석

### 왜 이전에는 여백이 생겼을까?
1. Home 컴포넌트는 `fixed inset-0`로 전체 화면 사용
2. 하지만 부모인 `#root`에 `padding: 2rem` 존재
3. Fixed 요소는 viewport 기준이므로 #root padding과 무관해야 하지만, 일부 브라우저에서 렌더링 이슈 발생 가능
4. `max-width: 1280px`로 인해 큰 화면에서 좌우 자동 마진 발생

### 해결 방법
- `#root`를 `width: 100%`, `height: 100vh`로 설정
- 모든 margin, padding 제거
- Fixed positioning이 올바르게 작동하도록 깔끔한 컨테이너 제공

## 안된 것
- 파비콘은 개발 서버 재시작 후 확인 필요
