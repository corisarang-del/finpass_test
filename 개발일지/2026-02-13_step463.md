# Step 463 개발일지

- 작성시각: 2026-02-13 18:22:43
- 해결하고자 한 문제:
  - 액션 상세 페이지에서 타이틀만 다르고 하위 데이터(실행 단계/차트 산식/표 수치)가 유사하게 보이던 문제를 해결.
  - `categoryId + actionIndex` 조합으로 액션 시나리오를 분리해 카테고리별 상세 차이를 명확히 반영.

- 해결된 것:
  - `ActionPlanDetail.tsx`에 `buildScenario(categoryId, actionIndex, ...)` 2차 매핑 로직 적용.
  - 카테고리(`real-estate`, `insurance`, `stock`, `life-balance`)별 1/2/3 액션 각각에 대해:
    - 서로 다른 실행 단계(steps)
    - 서로 다른 계산계수(`yearlyMultiplier`, `yearlyGrowthBonus`, `progressDelta`)
    - 서로 다른 표 항목(`tableRows`) 반영
  - 기본 표에 공통값(현재/실행 후 월 저축액)을 유지하면서, 시나리오별 상세 행을 추가하도록 확장.
  - `pnpm eslint`(cmd 경유)로 수정 파일 린트 통과 확인.

- 해결되지 않은 것:
  - 실제 금융상품/세율/시장데이터 기반 정밀 추정은 아직 미적용.
  - 카테고리별 표 항목 문구는 현재 규칙 기반이며, 운영 데이터 연동은 후속 작업 필요.
