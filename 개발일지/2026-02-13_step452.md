# 개발일지 Step 452
- 작성시각: 2026-02-13 16:25

## 해결하려던 문제
- `Result` 화면에서 희망 월 생활비 버튼이 눌려도 시각적 반응이 거의 없고, 페이지 스타일이 의도와 다르게 단순하게 보이던 문제
- `Result`, `LiveSimulation`, `FinalAnalysis`가 업로드된 목표 UI 흐름(시뮬레이션 카드/액션 플랜 카드)에 비해 디자인 톤이 맞지 않던 문제
- 화면이 길어질 때 스크롤이 막혀 하단 콘텐츠 확인이 어려운 문제

## 해결된 것
- 전역 스크롤 가능하도록 `src/index.css`의 `html/body` overflow를 `overflow-y: auto`로 변경함
- `Result.tsx`를 흰 배경 기반 + 가이드 버튼 스타일(선택 시 핑크 강조)로 재구성함
- `LiveSimulation.tsx`를 카드 중심 레이아웃(예상 은퇴 나이, 핵심 지표, 지출 조정, 요약)으로 재구성함
- `FinalAnalysis.tsx`를 액션 플랜 카드 중심 레이아웃으로 재구성함
- 세 페이지 모두 한 화면에 다 안 보이면 아래로 스크롤해서 확인 가능하게 구성함
- 수정한 TSX 파일 eslint 통과 확인함

## 해결 안 된 것
- 프로젝트 전체 `pnpm build` 실패(기존 다른 파일의 미사용 변수/타입 문제)는 이번 범위에서 처리하지 않음
