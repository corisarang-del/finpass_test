# 개발일지 - 2026-02-11 (Unsplash 이미지를 로컬 이미지로 교체)

작성시각: 2026-02-11 17:46

## 해결하고자 한 문제
1. Unsplash 외부 이미지 URL 사용으로 인한 로딩 속도 및 PS 로고 이슈
2. 로컬 이미지 파일을 사용하여 성능 개선 및 완전한 오프라인 지원

## 해결된 것
### 1. 이미지 파일 확인
`src/assets/images` 폴더에 총 7개 이미지 파일 존재:
- **홈화면 배경**: home1.png, home2.png, home3.png
- **카테고리 배경**: 기초자산.png, 라이프.png, 부동산.png, 주식.png

### 2. Home.tsx 수정
#### 이미지 import 추가 (8-14줄)
```tsx
import home1 from '../assets/images/home1.png';
import home2 from '../assets/images/home2.png';
import home3 from '../assets/images/home3.png';
import 부동산Image from '../assets/images/부동산.png';
import 기초자산Image from '../assets/images/기초자산.png';
import 주식Image from '../assets/images/주식.png';
import 라이프Image from '../assets/images/라이프.png';
```

#### 카테고리 이미지 경로 수정 (22-25줄)
```tsx
// 변경 전
image: 'https://images.unsplash.com/...'

// 변경 후
image: 부동산Image
image: 기초자산Image
image: 주식Image
image: 라이프Image
```

#### 홈화면 배경 이미지 경로 수정 (38-42줄)
```tsx
// 변경 전
bgImage: "https://images.unsplash.com/..."

// 변경 후
bgImage: home1
bgImage: home2
bgImage: home3
```

## 기술적 세부사항

### 장점
1. **성능 개선**: 외부 API 호출 없이 로컬 파일 사용으로 로딩 속도 향상
2. **오프라인 지원**: 인터넷 없이도 이미지 표시 가능
3. **빌드 최적화**: Vite가 이미지를 번들링하고 최적화
4. **안정성**: Unsplash API 장애나 rate limit 걱정 없음

### 이미지 처리 방식
- Vite는 import된 이미지를 자동으로 최적화하고 hash를 추가한 파일명으로 빌드
- 예: `부동산.png` → `/assets/부동산-abc123.png`
- 브라우저 캐싱에 유리
