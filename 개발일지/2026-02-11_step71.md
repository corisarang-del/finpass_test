# 개발일지 - 2026-02-11 (여백 문제 완전 해결 - Tailwind 오버라이드)

작성시각: 2026-02-11 17:34

## 해결하고자 한 문제
1. 개발 서버 재시작 후에도 카테고리 이미지에 여백이 남음
2. 포토샵 아이콘 작업 표시줄에서 제거하고 싶음

## 해결된 것
### 1. 여백 문제의 진짜 원인 발견
Tailwind CSS의 `@tailwind base`가 기본 스타일을 적용하면서 우리가 설정한 margin/padding을 덮어쓸 수 있었음.

#### index.css 수정
```css
// 변경 전
body {
    margin: 0;
    padding: 0;
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
}

// 변경 후
html {
    margin: 0;
    padding: 0;
    width: 100%;
    height: 100%;
    overflow: hidden;
}

body {
    margin: 0;
    padding: 0;
    width: 100%;
    height: 100%;
    overflow: hidden;
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
}
```

#### App.css 수정
```css
#root {
  width: 100%;
  height: 100vh;
  margin: 0;
  padding: 0;
  overflow: hidden; // ← 추가
}
```

### 2. 포토샵 작업 표시줄 제거 방법
1. **작업 표시줄의 Adobe Photoshop 아이콘 우클릭**
2. **"작업 표시줄에서 고정 해제" 선택**
또는
1. **포토샵 완전 종료** (작업 표시줄 아이콘 우클릭 → "창 닫기")

## 기술적 세부사항

### Tailwind CSS와의 충돌
- Tailwind의 `@tailwind base`는 Preflight라는 CSS 리셋을 포함
- 이것이 우리의 커스텀 margin/padding 설정을 덮어쓸 수 있음
- 해결: Tailwind 로드 후 명시적으로 html, body 스타일 재정의

### Overflow Hidden의 중요성
- `overflow: hidden`을 추가하여 의도하지 않은 스크롤바 제거
- html, body, #root 모두에 적용하여 완벽한 fullscreen 보장

### 왜 이렇게 해야 하는가?
1. html, body가 100% 크기가 아니면 vh 단위가 제대로 작동하지 않을 수 있음
2. Tailwind base가 기본 여백을 설정할 수 있음
3. overflow hidden으로 스크롤 방지
