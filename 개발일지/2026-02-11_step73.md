# 개발일지 - 2026-02-11 (카테고리 여백 근본 해결)

작성시각: 2026-02-11 18:04

## 해결하고자 한 문제
1. 카테고리 이미지 사이에 회색 여백이 계속 남는 문제
2. Tailwind CSS 클래스와 Grid 레이아웃이 제대로 적용되지 않는 문제

## 해결된 것

### 근본 원인 분석
- `motion.button` 요소가 CSS Grid cell을 완전히 채우지 못함
- `button` 요소는 기본적으로 content 크기만큼만 차지하는 특성이 있음
- Tailwind CSS 클래스가 framer-motion 컴포넌트에서 제대로 작동하지 않을 수 있음

### 해결 방법: 인라인 스타일로 완전 교체

#### 1. 컨테이너 변경
```tsx
// 변경 전: Tailwind Grid
className="fixed inset-0 w-full h-full z-40 grid grid-cols-1 grid-rows-4 ..."

// 변경 후: 인라인 스타일 Flex
style={{
  position: 'fixed',
  top: 0, left: 0,
  width: '100vw',
  height: '100vh',
  display: 'flex',
  flexDirection: 'column',
  overflow: 'hidden',
}}
```

#### 2. 카테고리 카드 변경
```tsx
// 변경 전: motion.button + Tailwind
<motion.button className="group relative w-full h-full overflow-hidden ...">

// 변경 후: motion.div + 인라인 스타일
<motion.div
  style={{
    position: 'relative',
    width: '100%',
    height: '25vh',  // ← 정확히 25% 높이
    overflow: 'hidden',
    cursor: 'pointer',
    margin: 0,
    padding: 0,
    border: 'none',
  }}
  className="group"
>
```

#### 3. 이미지도 인라인 스타일
```tsx
style={{
  position: 'absolute',
  top: 0, left: 0,
  width: '100%',
  height: '100%',
  objectFit: 'cover',
  display: 'block',
}}
```

## 기술적 세부사항

### 왜 인라인 스타일인가?
1. **Tailwind CSS 우선순위 문제**: Tailwind 클래스가 framer-motion과 충돌할 수 있음
2. **브라우저 기본 스타일 무시**: button 요소의 기본 스타일이 레이아웃에 영향
3. **확실한 적용**: 인라인 스타일은 최고 우선순위를 가짐
4. **height: 25vh**: 4개 카테고리가 정확히 화면을 4등분

### button → div 변경 이유
- `button` 요소는 기본적으로 `display: inline-block`이고 content 크기에 맞춰짐
- Grid/Flex에서 stretch가 안 될 수 있음
- `div`는 `display: block`이라 부모 크기에 맞춰 stretch됨
- 접근성을 위해 `cursor: pointer`와 `onClick` 유지
